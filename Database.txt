CREATE TABLE `book` (
  `name` varchar(30) NOT NULL,
  `location` varchar(45) NOT NULL,
  `status` varchar(30) NOT NULL DEFAULT 'Available',
  `sid` int DEFAULT NULL,
  `bid` int NOT NULL AUTO_INCREMENT,
  PRIMARY KEY (`bid`)
) ENGINE=InnoDB AUTO_INCREMENT=33 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci


CREATE TABLE `student` (
  `sid` int NOT NULL AUTO_INCREMENT,
  `name` varchar(45) NOT NULL,
  `stdate` varchar(45) NOT NULL,
  `returndate` varchar(45) NOT NULL,
  `phno` varchar(45) NOT NULL,
  PRIMARY KEY (`sid`)
) ENGINE=InnoDB AUTO_INCREMENT=36 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci

Abstract of database design:
Book(bid,bname,location,status,sid)-bid is primary key,this sid is referring to sid att in student table(FD:bid->bname,location,status,sid)
student(sid,sname,stdate,retdate,phno)-sid is primary key(FD:sid->sname,stdate,retdate,phno)

Process:Initial table is single table of Book(bid,bname,location,status,sname,stdate,retdate,phno) without sid attribute(book and student details)
later thought to include sid new attribute and made FD of sid->sname,stdate,retdate,phno possible to apply
and this is transitive dependency of non key determining non keys violating 3nf so decomposed to two tables appropriately
